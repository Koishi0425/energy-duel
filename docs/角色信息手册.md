## 内容维护分工

为了让长篇说明适合维护、短篇教程适合在游戏内阅读，玩家内容分成三层：

1. `shared/config/game.json` 是可执行规则的唯一来源，保存招式名称、费用、等级、速度、目标、解锁条件与效果描述。
2. `client/src/content/gameGuide.ts` 保存游戏内帮助中心的阅读结构、角色定位、对局思路和术语解释。引用角色或招式时只写稳定 ID，不重复维护招式数值。
3. `docs/` 保存完整规则、结算边界与设计定位，适合开发核对和深入阅读，不在浏览器运行时解析 Markdown。

新增角色时，应先完成游戏配置，再补充角色 Markdown 和 `characterGuides` 条目。游戏内教程会从配置读取完整技能树与数值，并由测试检查角色和招式引用是否缺失。
