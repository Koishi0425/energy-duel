# 技能伤害结算规则说明书

> 状态：权威规则规范
>
> 本文使用受控自然语言。`必须`表示强制要求，`不得`表示禁止要求，`可以`表示许可要求。
> 未在本文中定义的角色专属例外必须由角色规则明确声明；角色专属例外只覆盖其明确声明的条款。

## 1. 适用范围与优先级

1.1 本规则用于结算一个攻击技能对一个目标产生的附带效果、伤害、防御资源变化和健康状态变化。

1.2 一次伤害结算必须依次执行以下阶段：

1. 效果对抗阶段。
2. 伤害对抗阶段。
3. 伤害段防御阶段。
4. 回合健康结算阶段。

1.3 速度、目标有效性、攻击范围、锁定、位移和效果队列必须先按《基础规则手册》确定。只有对当前攻击者有效且满足速度条件的目标技能，才是本规则所称的对应技能。双方伤害等级相同时忽略速度门槛的既有规则仍然适用。

1.4 防御类行动不得作为对应技能参加效果对抗或伤害对抗。防御类行动可以在伤害段防御阶段提供格挡。

1.5 若角色规则明确声明通用规则的例外，则系统必须只在该角色规则声明的条件和范围内应用例外。

## 2. 术语

2.1 **攻击者**是发起当前攻击效果的单位。

2.2 **目标**是当前攻击效果准备影响的单位。

2.3 **对应技能**是由目标提交、对攻击者有效并且满足对抗速度条件的非防御技能。

2.4 **效果等级**只用于确定技能附带效果是否成功。配置中的 `effectLevel` 是效果等级；未声明 `effectLevel` 时，`level` 是效果等级。

2.5 **附带效果**是技能明确声明的非伤害结果，包括施加 Buff、取消行动、改变资源或改变其他规则状态。附带效果不得仅因伤害结果成功而生效。

2.6 **效果差**是当前技能效果等级减去对应技能效果等级所得的数值。

2.7 **效果结果**是“附带效果成功”或“附带效果失败”。效果差大于或等于 0.5 时，当前技能的附带效果成功；效果差小于 0.5 时，当前技能的附带效果失败。

2.8 **伤害等级**只用于确定一个伤害段的伤害结果。配置中的 `damageLevel` 是伤害等级；未声明 `damageLevel` 时，伤害等级继承效果等级。

2.9 **原伤害等级**是一个伤害段进入伤害对抗和防御判定前的伤害等级。系统必须保留原伤害等级，直至本回合健康结算完成。

2.10 **伤害差**是当前伤害段的原伤害等级减去对应技能的单段伤害等级所得的数值。

2.11 **伤害结果**是“产生伤害”或“不产生伤害”。伤害差大于 0 时产生伤害；伤害差小于或等于 0 时不产生伤害。

2.12 **剩余伤害等级**是产生伤害的伤害段经过伤害对抗和防御判定后剩余的伤害等级。

2.13 **伤害段**是独立执行伤害对抗、消耗防御资源并产生一个健康候选值的最小攻击单位。

2.14 **多段攻击**是包含两个或更多伤害段的攻击技能。

2.15 **生成性格挡**是由当前行动或临时机制重新生成的格挡。生成性格挡可以被本次攻击判定为破碎，但该破碎不得改变下一次重新生成的格挡等级。

2.16 **持续性格挡**是破碎状态会持续存在的格挡。持续性格挡破碎后，后续伤害段必须读取其更新后的零级格挡。

2.17 **健康状态左移**表示健康状态向死亡方向移动一个状态格。

## 3. 结算输入

3.1 每次伤害结算必须至少读取以下数据：

1. 攻击者和目标。
2. 当前技能的效果等级、伤害等级、有效速度和伤害段集合。
3. 每个伤害段的原伤害等级和伤害类型。
4. 对应技能是否存在。
5. 对应技能的效果等级和单段伤害等级。
6. 当前技能明确声明的附带效果。
7. 目标当前的屏障、神体、格挡和护甲数据。
8. 目标本回合已经登记的攻击伤害健康候选值。

3.2 伤害类型必须是普通、穿刺或真实之一。未显式声明穿刺或真实的伤害必须按普通伤害结算。

## 4. 效果对抗与伤害对抗

4.1 系统必须分别计算效果结果和伤害结果。任一结果不得作为另一结果的前置条件。

### 4.1 效果对抗

4.2 若不存在对应技能，则当前技能的附带效果成功。

4.3 若存在对应技能，则系统必须计算效果差。

4.4 若效果差大于或等于 0.5，则当前技能的附带效果成功。

4.5 若效果差小于 0.5，则当前技能的附带效果失败。系统不得应用该技能中受效果等级影响的附带效果。

4.6 附带效果成功不得直接视为产生伤害。当前技能的伤害等级小于或等于对应技能的伤害等级时，附带效果仍然可以成功。

### 4.2 伤害对抗

4.7 若不存在对应技能，则每个伤害段的伤害差必须等于该段的原伤害等级。

4.8 若存在对应技能，则每个伤害段必须独立按下式计算伤害差：

```text
伤害差 = 当前伤害段的原伤害等级 - 对应技能的单段伤害等级
```

4.9 若伤害差大于 0，则该伤害段产生伤害，并且进入防御判定前的剩余伤害等级必须等于伤害差。

4.10 若伤害差小于或等于 0，则该伤害段不产生伤害。系统不得令该伤害段消耗防御资源或登记健康候选值。

4.11 伤害结果不得读取效果结果。当前技能的效果差小于 0.5 时，只要伤害差大于 0，该技能仍然产生伤害。

4.12 对应技能必须在其自身的效果队列位置结算。一次对抗不得令对应技能提前执行或重复执行。

## 5. 多段攻击

5.1 每次目标分配必须生成一个伤害段。允许多个伤害段指向同一目标。

5.2 若目标具有对应攻击技能，则指向该目标的全部伤害段必须只合并效果等级，并执行一次效果对抗。

5.3 合并效果等级必须按下式计算：

```text
合并效果等级 = 单段效果等级 x 指向该目标的伤害段数量
```

5.4 伤害段的原伤害等级不得相加。无论效果结果为何，每个伤害段都必须按原有顺序独立执行伤害对抗；只有伤害差大于 0 的伤害段进入完整防御判定。

5.5 屏障、持续性格挡、护甲和其他可消耗防御资源发生变化后，后续伤害段必须读取更新后的数值。

5.6 合并后的效果结果必须共同用于该次多段攻击明确声明的附带效果。该效果结果不得改变任一伤害段的伤害结果。

5.7 每个剩余伤害等级大于 0 的伤害段必须独立登记健康候选值；健康状态的最终变化仍必须遵守第 7 节的回合上限。

## 6. 伤害段防御

6.1 每个伤害段必须按以下固定顺序执行防御判定：

1. 屏障。
2. 神体。
3. 格挡。
4. 护甲。

6.2 任一防御步骤使剩余伤害等级小于或等于 0 时，系统必须结束当前伤害段；系统不得继续执行该段后续防御或健康候选登记。

### 6.1 屏障

6.3 若目标具有可触发的屏障，则系统必须消耗一层屏障，并将当前伤害段的剩余伤害等级设为 0。

6.4 屏障必须能够抵消普通、穿刺和真实伤害。每个伤害段最多消耗一层屏障。

6.5 泥岩屏障是角色专属例外：泥岩本回合使用防御技能时，该屏障不得触发；该屏障触发时必须额外使泥岩的健康状态右移一格。

### 6.2 神体

6.6 普通伤害和穿刺伤害必须减去目标的神体等级。真实伤害必须跳过神体判定。

### 6.3 格挡

6.7 普通伤害和真实伤害必须减去目标的有效格挡等级。穿刺伤害必须跳过格挡判定。

6.8 对可破碎格挡，若进入格挡判定时的剩余伤害等级大于或等于格挡等级，则系统必须判定该格挡破碎。

6.9 当前伤害段必须先使用破碎前的完整格挡等级。格挡是否破碎不得改变本段的减伤值。

### 6.4 护甲

6.10 只有普通伤害可以消耗护甲。穿刺伤害和真实伤害必须跳过护甲判定。

6.11 护甲抵消值必须按下式计算：

```text
护甲抵消值 = min(当前剩余伤害等级, 当前护甲等级)
```

6.12 系统必须同时从剩余伤害等级和护甲等级中减去护甲抵消值。

## 7. 回合健康结算

7.1 每个剩余伤害等级大于 0 的伤害段必须登记一个健康候选值。防御资源变化必须立即保留；健康状态不得简单累加每个伤害段的左移次数。

7.2 系统必须保留目标本回合收到的最高剩余伤害等级，以及是否存在一个“原伤害等级至少为 3 且剩余伤害等级至少为 1”的重击候选。

7.3 若后出现的候选值高于先前候选值，系统必须以本回合开始时的健康状态为基准重新计算攻击伤害造成的健康结果。系统不得在已经应用的较低候选结果上继续累加。

7.4 若本回合不存在重击候选，则所有攻击伤害合计最多使目标健康状态左移一格。

7.5 若最高剩余伤害等级小于 0.5，则攻击伤害不得改变目标健康状态。

7.6 若最高剩余伤害等级大于或等于 0.5，且本回合不存在重击候选，则攻击伤害必须使目标健康状态左移一格。

7.7 若本回合存在重击候选，则攻击伤害必须使目标健康状态从本回合开始时的状态左移两格。

7.8 基础角色只有健康和死亡两个状态。对基础角色应用一次左移必须使其进入死亡状态。

7.9 脆弱、泥岩出拳风险、内卫装置锁定、超脱复生和无忧觉等角色规则必须在第 7.4 至 7.7 条所得结果上应用各自明确声明的例外。

7.10 直接声明“左移健康状态”的效果不是攻击伤害。该效果必须立即结算，并且不得计入本节的最高剩余伤害等级或每回合攻击左移上限。

## 8. 形式化流程

```text
结算攻击(攻击者, 目标, 技能):
  对应技能 := 读取满足目标、范围和速度条件的目标技能
  效果结果 := 独立比较效果等级
  若效果结果成功，则应用技能明确声明的附带效果
  伤害段集合 := 按原顺序生成；不得合并伤害等级
  对每个伤害段:
    伤害结果 := 独立比较双方单段伤害等级
    若伤害差小于或等于 0，则跳过该伤害段
    若目标已不具备继续承受伤害的条件，则按退场或复生规则处理
    依次执行屏障、神体、格挡、护甲
    若剩余伤害等级大于 0，则登记健康候选值
  依据本回合最高候选与重击候选更新健康状态
```

## 9. 验收矩阵

| 场景 | 必须结果 |
| --- | --- |
| 当前技能效果 2、伤害 0.5；对应技能效果 1.5、伤害 1.5 | 当前技能的附带效果成功；当前技能不产生伤害 |
| 当前技能效果 1.5、伤害 1.5；对应技能效果 2、伤害 0.5 | 当前技能的附带效果失败；当前技能以 1 级伤害差进入防御结算 |
| 当前技能效果 2、伤害 1；对应技能效果 1.75、伤害 0 | 效果差只有 0.25，附带效果失败；当前技能仍以 1 级伤害差进入防御结算 |
| 双方效果等级相等、伤害等级不等 | 双方附带效果均失败；伤害等级较高的一方仍可产生伤害 |
| 2.5 原伤害，0.5 格挡 | 剩余 2；本回合无重击候选时左移 1 格 |
| 3 原伤害，2 格挡 | 剩余 1；登记重击候选；从回合开始状态左移 2 格 |
| 3 原伤害，0.5 神体，2 格挡 | 剩余 0.5；不得登记重击候选；本回合攻击合计最多左移 1 格 |
| 真实伤害，目标有神体和格挡 | 跳过神体；正常计算格挡 |
| 穿刺伤害，目标有神体、格挡和护甲 | 计算神体；跳过格挡和护甲 |
| 任意伤害，目标有可触发屏障 | 屏障先消耗；该伤害段不得消耗神体、格挡或护甲 |
| 两个普通伤害段均为 1.5 | 两段独立消耗防御；本回合健康状态合计最多左移 1 格 |
| 先收到 0.5 剩余伤害，后收到满足重击条件的伤害 | 后者替换前者的健康候选；从回合开始状态合计左移 2 格 |
