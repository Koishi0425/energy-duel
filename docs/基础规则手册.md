# 《娇斯拉大战贡刚》基础规则手册

> 版本：v5（依据 `基础规则说明书.xlsx` 与当前可执行规则整理）
>
> 游戏实际结算以 `shared/config/game.json` 与服务端代码为准。本手册用于解释规则，不作为可执行配置。

## 1. 游戏目标

所有玩家同时选择行动，服务端按速度和稳定 ID 顺序生成结算队列。存活到最后的一名玩家获胜；若同一回合无人存活，则本局平局。

## 2. 回合流程

1. **选择阶段**：存活玩家选择一个行动，并在需要时预先选择目标。
2. **等待阶段**：行动内容对其他玩家保密，只公开“已提交”。最后一名存活玩家提交前，玩家可以撤销并重选。
3. **后发阶段**：若本回合有后发技能，服务端公开所有行动并依次收集目标；允许保留的鬼影冲刺可以跳过本回合。
4. **结算阶段**：目标锁定后，服务端按速度从高到低、玩家 ID 从小到大执行效果；位移、治疗等较快效果会真实改变后续技能的结果，客户端按同一队列播放时间线。
5. **新回合或结束**：服务端统一应用结果。只剩一名玩家时结束游戏；所有在房玩家确认结算后，房间返回准备阶段，可以继续下一局。每回合所有行动（包括没有获得资源、没有受到攻击等未产生实际效果的行动）都会写入带局数、回合和时间的战斗日志，日志直接使用健康、濒死或死亡等状态名称。

## 3. 行动分类

- **基础**：初始角色始终可用的气、蓄力、凹、紫翼双凹和剁。
- **攻击**：对目标造成健康状态变化，如拳、斩和角色专属攻击。
- **防御**：提供本回合格挡或免疫，如挡、斧挡和超防。
- **资源**：为未来独立的资源类招式预留；当前初始资源招式归入“基础”。
- **特殊**：治疗、变身或建立角色状态，如治疗和举斧。

原表中的“做功 / 非做功”不参与当前规则、界面分类或结算。

## 4. 速度、等级与生命

- **速度**决定权威结算与动作时间线顺序，数值越高越先执行；速度相同时按稳定玩家 ID 排序。
- **等级**用于招式对抗，不再与速度相加。受到攻击时，以“受到的攻击等级 − 自己本回合对该攻击者有效的招式等级”计算等级差。防御和无需选择目标的行动对所有来袭攻击有效；需要选择玩家、路径或范围的行动，只有确实把该攻击者包含在目标或作用范围内时才提供等级。例如 A 攻击 B、C 攻击 A 时，A 对 B 使用的招式等级不能用于抵挡 C。
- 变身前角色只有“健康 / 死亡”两个有效状态，因此受到足以改变健康状态的伤害会直接淘汰。
- 变身后角色解锁“健康 / 濒死 / 死亡”：从健康进入濒死时获得 1 气；再次受伤则死亡。
- 等级差小于 0.5 时攻击被抵消；等级差大于等于 0.5 且小于 1 时，健康状态左移一层；等级差大于等于 1 时直接死亡。
- 伤害类技能等级低于 3 时，即使等级差更高也最多只左移一个健康状态；3 级及以上攻击才会触发跨状态直接死亡。
- 受伤类效果称为**左移**，治疗类效果称为**右移**。
- **脆弱**：处于脆弱行动期间，受到任何有效伤害或健康状态左移都会死亡。

## 5. 目标选择与“后发”

普通招式的目标在选择阶段确定：选择 1 名其他存活玩家的行动点选一次；紫翼双凹选择 2 次，可以重复选择同一名玩家。

储君的星尘、李淳罡的剑气/一剑开天门、熟练二层后的吸以及梦魇释放时的鬼影冲刺使用后发目标时机。所有存活玩家提交后，服务端公开本回合行动，再要求对应玩家选择目标。星尘会固定消耗释放时持有的全部辉星，并按消耗数量进行可重复分配；鬼影重重可以跳过当回合并把唯一一次冲刺保留到下一回合。所有后发选择完成后才统一结算。

## 6. 圆形地块与位移

- 棋盘始终有 `2 × 玩家数` 个地块；索引从右侧 0 开始，沿屏幕顺时针增加。
- 地块编号始终显示。选择地块的技能会直接高亮合法地块，玩家点击地图上的高亮地块完成选择。
- 一次普通位移只能选择顺时针或逆时针相邻空格，不能与其他存活玩家重叠。多人同时争夺同一空格时，按权威结算顺序先到者占据，后到者留在原地。
- 每局开始以及上一局结算确认完成后，玩家位置都会按加入顺序重置到 `0、2、4……` 的初始地块。
- 范围和路径技能在自身效果执行时读取最新权威位置，因此速度更高的位移可以躲开、改变或承接之后的传导攻击。
- 伤害类型目前用于技能与被动的明确交互；没有说明时，钝击、斩击、魔法与普通攻击仍使用相同的等级差生命规则。

资源可以出现分数。李淳罡的斩支付 `1/3` 气；界面和线协议保留该精度。标记“任意资源”的技能只接受整数点数，由玩家在提交时选择支付组成；不足 1 点的分数余额不能单独支付，服务端会校验整数、各资源余额与总和。气和蓄力始终显示；当前角色拥有的特殊资源始终显示；其他角色的特殊资源仅在数值大于 0 时显示。

## 7. 基础招式

| 分类 | 招式 | 消耗 | 速度 | 等级 | 目标或作用对象 | 规则摘要 |
| --- | --- | ---: | ---: | ---: | --- | --- |
| 基础 | 气 | 无 | 1 | 0 | 自身（自动） | 获得 1 气。 |
| 基础 | 蓄力 | 无 | 1 | 0 | 自身（自动） | 获得 1 蓄力。 |
| 攻击 | 拳 | 无（需持有至少 1 气才解锁） | 1 | 0.5 | 1 名其他存活玩家（预选） | 0.5 级钝击；出招期间脆弱。 |
| 攻击 | 斩 | 1 气 | 1 | 1.5 | 1 名其他存活玩家（预选） | 1.5 级斩击。 |
| 防御 | 挡 | 无 | 2 | 2 | 自身（自动） | 格挡低于 2 级的攻击；达到破防条件后不再格挡。 |
| 基础 | 凹 | 无 | 1 | 0 | 1 名其他存活玩家（预选） | 若该玩家本回合出气，偷取该气。 |
| 基础 | 紫翼双凹 | 无 | 1 | 0 | 其他存活玩家，选择 2 次（预选，可重复） | 每次按凹结算；出招期间脆弱。 |
| 基础 | 剁 | 无 | 1 | 0 | 本回合使用凹类技能的所有玩家（自动） | 取消普通凹、紫翼双凹及后续凹类技能；无视双方等级，使每名使用者损失一个健康状态。 |
| 防御 | 超防 | 1 气 | 4 | ∞ | 自身（自动） | 本回合免疫任何攻击。 |
| 特殊 | 治疗 | 无 | 0 | 0 | 自身（自动） | 恢复一个健康状态；出招期间脆弱。 |
| 特殊 | 变身 | 由所选角色决定 | 4 | 0 | 1 个非当前角色（预选） | 切换角色并加载其技能树；当前所有角色均免费。 |

初始角色可以使用五个基础招式、挡、超防和变身，但不能直接使用拳、斩或治疗。完成变身后仍保留这些已有招式，并解锁所选角色配置中的攻击、防御和特殊招式。

变身没有每局一次的限制。玩家可以在之后的回合继续切换为当前角色以外的其他角色。默认情况下 Buff 归属于产生它的角色：切走时暂不显示和生效，切回时恢复；有限持续时间在后台继续减少，不会因为角色不在场而冻结。配置明确标记为玩家级的 Buff 才会跨角色持续显示和生效。

## 8. 房间与连接

- 房主创建 4–10 位字母或数字房间号，其他玩家可直接输入加入。
- 所有人准备后，只有房主可以开始游戏。
- 等待阶段断开连接视为直接离开；游戏中意外断线保留席位 30 秒。
- 房主永久离开会销毁房间；游戏中其他玩家永久退出会结束当前对局。
- 账号使用唯一用户名与至少 7 位密码注册和登录；这是朋友间游玩的轻量账号系统，不应复用重要密码。
