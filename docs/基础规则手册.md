# 《娇斯拉大战贡刚》基础规则手册

> 版本：v3（依据 `基础规则说明书.xlsx` 与当前可执行规则整理）  
> 游戏实际结算以 `shared/config/game.json` 与服务端代码为准。本手册用于解释规则，不作为可执行配置。

## 1. 游戏目标

所有玩家同时选择行动，服务端按速度和稳定 ID 顺序生成结算队列。存活到最后的一名玩家获胜；若同一回合无人存活，则本局平局。

## 2. 回合流程

1. **选择阶段**：存活玩家选择一个行动，并在需要时预先选择目标。
2. **等待阶段**：行动内容对其他玩家保密，只公开“已提交”。最后一名存活玩家提交前，玩家可以撤销并重选。
3. **结算阶段**：最后一人提交后立即锁定行动。客户端按速度从高到低、玩家 ID 从小到大播放动作时间线；互相出招或“攻击与目标的防御”等相关动作会分组同时展示。
4. **新回合或结束**：服务端统一应用结果。只剩一名玩家时结束游戏；所有在房玩家确认结算后，房间返回准备阶段，可以继续下一局。每回合所有行动（包括没有获得资源、没有受到攻击等未产生实际效果的行动）都会写入带局数、回合和时间的战斗日志，日志直接使用健康、濒死或死亡等状态名称。

## 3. 行动分类

- **基础**：初始角色始终可用的气、蓄力、凹、紫翼双凹和剁。
- **攻击**：对目标造成健康状态变化，如拳、斩和角色专属攻击。
- **防御**：提供本回合格挡或免疫，如挡、斧挡和超防。
- **资源**：为未来独立的资源类招式预留；当前初始资源招式归入“基础”。
- **特殊**：治疗、变身或建立角色状态，如治疗和举斧。

原表中的“做功 / 非做功”不参与当前规则、界面分类或结算。

## 4. 速度、等级与生命

- **速度**决定动作时间线顺序，数值越高越先展示；速度相同时按稳定玩家 ID 排序。
- **等级**用于招式对抗，不再与速度相加。受到攻击时，以“受到的攻击等级 − 自己本回合招式等级”计算等级差；自己的招式可以是攻击、防御或其他行动。
- 变身前角色只有“健康 / 死亡”两个有效状态，因此受到足以改变健康状态的伤害会直接淘汰。
- 变身后角色解锁“健康 / 濒死 / 死亡”：从健康进入濒死时获得 1 气；再次受伤则死亡。
- 等级差小于 0.5 时攻击被抵消；等级差大于等于 0.5 且小于 1 时，健康状态左移一层；等级差大于等于 1 时直接死亡。
- 伤害类技能等级低于 3 时，即使等级差更高也最多只左移一个健康状态；3 级及以上攻击才会触发跨状态直接死亡。
- 受伤类效果称为**左移**，治疗类效果称为**右移**。
- **脆弱**：处于脆弱行动期间，受到任何有效伤害或健康状态左移都会死亡。

## 5. 目标选择与“后发”

普通招式的目标都在选择阶段预先确定：单目标行动点选一次；紫翼双凹点选两次，可重复选择同一目标。

储君的“星尘”启用“后发”目标时机。使用者在选择阶段先确定正整数 `n` 并支付 `n` 辉星；所有存活玩家提交后，服务端公开本回合行动，再要求星尘使用者分配 `n` 个 0.5 级攻击。可以重复选择同一目标，其攻击等级会在该目标上累加。所有需要后发选择的玩家完成分配后才进入统一结算。

## 6. 基础招式

| 分类 | 招式 | 消耗 | 速度 | 等级 | 规则摘要 |
| --- | --- | ---: | ---: | ---: | --- |
| 基础 | 气 | 无 | 1 | 0 | 获得 1 气。 |
| 基础 | 蓄力 | 无 | 1 | 0 | 获得 1 蓄力。 |
| 攻击 | 拳 | 无 | 1 | 0.5 | 单目标钝击；出招期间脆弱。 |
| 攻击 | 斩 | 1 气 | 1 | 1.5 | 单目标斩击。 |
| 防御 | 挡 | 无 | 2 | 2 | 格挡低于 2 级的攻击；达到破防条件后不再格挡。 |
| 基础 | 凹 | 无 | 1 | 0 | 若预选目标本回合出气，偷取该气。 |
| 基础 | 紫翼双凹 | 无 | 1 | 0 | 选择两次目标，可重复；出招期间脆弱。 |
| 基础 | 剁 | 无 | 1 | 0 | 取消全场凹，并使使用凹的玩家损失一个健康状态。 |
| 防御 | 超防 | 1 气 | 4 | ∞ | 本回合免疫任何攻击。 |
| 特殊 | 治疗 | 无 | 0 | 0 | 恢复一个健康状态；出招期间脆弱。 |
| 特殊 | 变身 | 由目标角色决定 | 4 | 0 | 切换到当前角色以外的其他角色，并加载其技能树。娇斯拉、贡刚与储君当前均为免费。 |

初始角色可以使用五个基础招式、挡、超防和变身，但不能直接使用拳、斩或治疗。完成变身后仍保留这些已有招式，并解锁所选角色配置中的攻击、防御和特殊招式。

变身没有每局一次的限制。玩家可以在之后的回合继续切换为当前角色以外的其他角色。默认情况下 Buff 归属于产生它的角色：切走时暂不显示和生效，切回时恢复；有限持续时间在后台继续减少，不会因为角色不在场而冻结。配置明确标记为玩家级的 Buff 才会跨角色持续显示和生效。

## 7. 房间与连接

- 房主创建 4–10 位字母或数字房间号，其他玩家可直接输入加入。
- 所有人准备后，只有房主可以开始游戏。
- 等待阶段断开连接视为直接离开；游戏中意外断线保留席位 30 秒。
- 房主永久离开会销毁房间；游戏中其他玩家永久退出会结束当前对局。
- 用户名登录没有密码保护，用户名只用于区分账号数据，不代表安全身份认证。
