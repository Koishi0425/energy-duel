# Buff 与状态速查

> 版本：v16（统一表结构，依据 `shared/config/game.json` 及各角色文档整理）
>
> 本表以**本体类型**为第一分类键，将 Buff、场地对象、资源、健康阶段、形态等所有"持续存在的事实"纳入同一框架。技能的**一次性结算效果**（伤害、资源增减等）不在此表中——它们属于过程而非持久事实。
>
> **作用域规则**：归属为「角色」的在切走角色时后台保留、持续时间继续扣除、切回时恢复；归属为「玩家」的跨角色持续生效；归属为「地块」的附着于棋盘位置。

## 表头说明

| 列 | 说明 |
|---|---|
| **本体类型** | 该事实在游戏逻辑中的种类，见下方枚举 |
| **来源** | 哪个技能/被动/系统产生此事实 |
| **施加时机/条件** | 在结算流程的哪个节点、满足什么条件时施加 |
| **目标** | 施加到谁身上 |
| **作用域** | 角色 / 玩家 / 地块 |
| **持续** | 持续类型 + 具体回合数（含当前则注明） |
| **层数/刷新** | 是否有层数、上限、重复施加时的行为 |
| **消除条件** | 除了自然到期外如何移除 |
| **具体效果** | 精确的效果描述，包含数值修正、授予/禁止技能等 |
| **可见性** | 公开（所有玩家可见）/ 仅己方 / 条件遮蔽 |
| **备注** | 互斥、依赖、免疫、特殊规则 |

### 本体类型枚举

| 类型 | 定义 | 审核要点 |
|---|---|---|
| **资源** | 可增减、可用于支付的数值 | 初始值、获取途径、消耗出口 |
| **属性修正** | 改变等级/速度/消耗等数值 | 修正值是否在合法范围、是否可叠加 |
| **行为许可** | 授予技能使用权 | 是否与当前形态技能树冲突 |
| **行为禁止** | 禁止某类技能 | 是否产生无解回合 |
| **伤害响应** | 改变来袭伤害的处理方式 | 与同类叠加时的优先级 |
| **健康阶段** | 健康/濒死/死亡 | 进出条件是否闭环 |
| **形态** | 当前活跃角色/形态 | 切换时哪些事实保留 |
| **计数器** | 单向累积至阈值触发 | 期望触发回合数 |
| **冷却** | 技能再可用倒计时 | 进度来源是否明确 |
| **标记** | 记录一次性事件是否已发生 | 作为后续技能的 gate |
| **信息遮蔽** | 改变可见信息集合 | 遮蔽范围与解除条件 |
| **棋盘存在** | 地块上的非玩家实体 | 生命周期、共存规则 |
| **回合触发** | 每回合自动产生某种效果 | 触发内容与时机 |

一行可能兼属多个本体类型（如「恐惧」同时是行为禁止和属性修正），此时按最主导的类型归类并在效果列中注明其余。
